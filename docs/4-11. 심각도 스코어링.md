# 4-11. 심각도 스코어링

[4-10. 도출 스멜 - 심각도 표](./4-10.%20도출%20스멜%20-%20심각도%20표.md) 의 18개 스멜을
**영향도 × 수정난이도** 매트릭스에 배치해 착수 우선순위를 정한다. 4-10의 심각도(상/중/하)가
"이 문제가 얼마나 나쁜가"였다면, 이 문서의 목적은 "무엇을 먼저 고칠 것인가"다.

## 1. 채점 기준

| 축 | 상 | 중 | 하 |
|---|---|---|---|
| **영향도** — 방치 시 비용 | 여러 파일/서비스에 걸쳐 있거나 테스트·안전한 변경 자체를 막음 | 특정 서비스 내 반복, 유지보수 비용을 계속 늘림 | 가독성 위주, 동작·확장성에 미치는 영향 적음 |
| **수정난이도** — 고치는 데 드는 리스크·범위 | 여러 클래스에 걸친 재설계, 회귀 위험 큼, 다른 스멜과 얽혀 있음 | 여러 지점을 손대야 하지만 각각은 패턴화 가능 | 한두 지점만 기계적으로 고치면 됨(Rename, 상수 추출 등) |

## 2. 매트릭스 → 우선순위 규칙

| | 난이도 하 | 난이도 중 | 난이도 상 |
|---|:---:|:---:|:---:|
| **영향도 상** | P1 최우선 | P1 최우선 | P2 계획 후 착수 |
| **영향도 중** | P1 최우선 | P2 계획 후 착수 | P3 여유 있을 때 |
| **영향도 하** | P3 여유 있을 때 | P3 여유 있을 때 | P4 보류 |

- **P1 (최우선)**: 영향도 대비 수정 비용이 낮은 Quick Win — 지금 바로 착수.
- **P2 (계획 후 착수)**: 영향도는 크지만 범위가 넓어 사전 설계·테스트가 필요.
- **P3 (여유 있을 때)**: 지금 당장은 아니어도 되지만, 손대는 김에 같이 처리하면 이득.
- **P4 (보류)**: 영향도 대비 비용이 커서 우선순위 밖. 이번 코드베이스에는 해당 항목 없음.

## 5. 권장 착수 순서

1. **No Tests** — 다른 모든 항목의 선행 조건. 현재 동작을 특성화 테스트로 고정.
2. **Primitive Obsession(매직넘버 → enum)** — 영향도 상이면서 난이도가 중에 그치는 유일한 항목이고,
   이후 God Class·Long Method 분해 시 매직넘버 비교문을 enum 비교로 먼저 바꿔두면 다음 단계가 쉬워지는
   **기반 작업** 성격이 강하다.
3. **Tight Coupling 해소(DI 적용)** — 인터페이스 추출 + 생성자 주입. 이후 테스트 작성 시 mail/audit를
   가짜 객체로 대체할 수 있게 된다.
4. **P1의 나머지 Quick Win들**(요청 DTO, 감사 로그 중복, role 판정 중복, Feature Envy, 깊은 중첩) —
   서로 독립적이라 순서 상관없이 병행 가능.
5. **God Class / Long Method(`processApproval()`)** — 위 P1 작업(특히 매직넘버 enum화, DI, 감사 로그
   추출)이 끝난 뒤 착수하면 `processApproval()` 을 쪼개는 작업 자체의 난이도가 크게 낮아진다. 먼저
   손대면 여러 스멜이 뒤엉킨 상태에서 리팩토링해야 해 회귀 위험이 크다.
6. **메일 본문 중복(P2)** — God Class 분해와 자연스럽게 같이 진행되는 편이 효율적(메일 발송 책임을
   분리하는 김에 템플릿도 함께 정리).
7. **P3(Anemic Domain Model, 캡슐화 부재)** — 장기 과제. CLAUDE.md의 Rich Domain Model 방향과 맞닿아
   있어 가치는 크지만, 모든 setter 호출부에 영향을 주므로 위 항목들이 안정된 뒤 착수하는 편이 안전하다.
8. **나머지 P3(라벨 중복·파라미터 목록·네이밍·주석)** — 위 리팩토링 과정에서 자연히 함께 해소되는
   경우가 많아 별도 일정을 잡기보다 진행 중 손이 닿을 때 같이 정리.

**제외 항목(조용한 실패)**: 영향도는 중간 수준이지만 CLAUDE.md 불변 규칙상 리팩토링 단계에서 동작을
바꿀 수 없어 매트릭스 채점 자체가 무의미하다 — 특성화 테스트로 "지금 이렇게 동작한다"는 사실만 고정해
두고, 수정 여부는 별도 재설계 단계의 판단으로 넘긴다.
